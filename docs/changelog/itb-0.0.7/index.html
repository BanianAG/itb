ğŸ—ï¸ Architecture Updatesâ€‹

1. Traefik Load Balancer (NEW)â€‹

This release introduces Traefik as a local load balancer, providing a cleaner URL structure for development:

Local Access URLs:

  • WebApp: http://itb.localhost (instead of port-based URLs)
  • API: http://api.itb.localhost (instead of port-based URLs)
  • Traefik Dashboard: http://traefik.localhost:8080 (optional monitoring)

Why this helps:

  • Simplified development URLs
  • Closer alignment with production architecture
  • No need to remember port numbers
  • Supports multiple projects running simultaneously

Setup: Your /etc/hosts file should include (already configured if using provided docker-compose):

127.0.0.1 itb.localhost
127.0.0.1 api.itb.localhost
127.0.0.1 traefik.localhost

2. Integrated Documentation Container (NEW)â€‹

The ITB documentation is now packaged as an integrated container service running busybox with httpd webserver:

Access Documentation:

  • Access directly: http://itb.localhost/docs
  • Documentation is fully functional without external network calls

Benefits:

  • Documentation always matches your deployed version
  • Offline-friendly within your local/internal network
  • No external dependencies
  • Consistent documentation across teams

For Existing Usersâ€‹

Follow these steps to upgrade to ITB 0.0.7:

  1. Backup your data:

    # Backup the current model
    save remote in the WebApp to export the Model to JSON
    Backup your Output / JSON Folder

  2. Update your environment:

    # Stop current instance
    docker compose down

    # Pull latest itb version from GitHub
    git pull

    # Backup and update .env
    cp .env .env.backup
    cp .env.example .env
    # Re-apply your custom settings

  3. Pull latest images and start:

    docker compose pull
    docker compose up -d

  4. Verify the installation:

    • WebApp: http://itb.localhost
    • API: http://api.itb.localhost
    • Docs: http://itb.localhost/docs

ğŸ”§ Troubleshootingâ€‹

Issue: Containers not starting or old configuration persistsâ€‹

If you encounter issues after upgrading, try the following steps to perform a clean start:

Option 1: Clean Restart (Recommended)â€‹

Stop and remove all containers:

# Stop all running containers
docker compose down

# Verify no ITB containers are running
docker ps | grep itb
# (should return no results)

Remove the PocketBase data volume (âš ï¸ This will reset your database):

# List volumes to find pb_data
docker volume ls | grep pb_data

# Remove the volume
docker volume rm itb_pb_data
# (or adjust the name based on your project folder name)

Start fresh:

docker compose up -d

Option 2: If you need to preserve your dataâ€‹

Stop containers but keep volume:

docker compose down

Then restart with the existing data:

docker compose up -d

Common Issues & Solutionsâ€‹

IssueSolution
Traefik routes not workingEnsure /etc/hosts has 127.0.0.1 itb.localhost api.itb.localhost entries

Full Reset (Last Resort)â€‹

If issues persist, perform a complete reset:

# Stop everything
docker compose down

# Remove all ITB-related containers
docker ps -a | grep itb | awk '{print $1}' | xargs docker rm -f

# Remove volume
docker volume rm itb_pb_data

# Clean up any dangling images (optional)
docker image prune -a

# Start fresh
docker compose up -d

âš ï¸ Warning: This will permanently delete your PocketBase data. Make sure you've exported your models first!
